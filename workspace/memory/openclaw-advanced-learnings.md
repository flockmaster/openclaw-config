## 1. 记忆蒸馏

### 什么是记忆蒸馏？
将冗长的对话历史压缩成精简的知识摘要，保留关键决策和信息，丢弃冗余内容。

### 效果
- 上下文从17K降到9.3K
- 后续对话更高效，成本更低
- 不会丢失重要上下文

### 应用场景
- 长期项目维护
- 复杂任务的多轮对话
- Agent跨会话知识传递

---

## 2. Skill固化

### 核心思想
让Agent像面向对象编程一样继承知识——把成熟的工作流、经验固化成Skill，后续Agent可以直接继承使用。

### 价值
- 新Agent不需要从零学习
- 成熟SOP可以复用
- 企业知识资产化

### 我们已有的实践
- `HEARTBEAT.md` - 心跳任务固化
- `AGENTS.md` - 操作规则固化
- 各Agent的agentDir中的指令

### 可改进方向
- 把交流会准备流程固化成Skill
- 把产品需求→PRD流程固化

## 3. 模型降级策略

### 核心思想
不是所有任务都需要昂贵模型。简单任务用便宜模型，复杂任务才用高级模型。

### 配置示例
```json
{
  "model_routing": {
    "default": "claude-haiku-4-5",
    "complex_reasoning": "claude-opus-4",
    "code_generation": "deepseek-r1",
    "simple_tasks": "gemini-2.0-flash"
  }
}
```

### 我们的现状
- 主模型：glm-5（便宜）
- 复杂任务：qwen3-max / kimi-k2.5
- 编码任务：qwen3-coder-plus

### 优化空间
- 心跳检查用最小模型
- 简单问答用便宜模型
- 复杂推理才用高级模型

## 4. 多Agent协作三种模式

来自另一个视频：

### 线性流水线
```
需求 → PRD → 原型 → 开发 → 测试
```
串行执行，每一步输出是下一步输入。

### 依赖图并行
```
     ┌→ 前端开发 →┐
需求 →            → 集成测试
     └→ 后端开发 →┘
```
独立任务并行执行，最后汇总。

### 多Agent辩论
多个Agent对同一问题提出方案，辩论后综合决策。
