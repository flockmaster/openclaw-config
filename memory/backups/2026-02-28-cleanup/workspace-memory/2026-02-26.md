## 重要讨论：异步任务架构升级

### 问题
- 长任务阻塞主对话（如下载模型时无法及时回复用户）
- 轮询任务状态浪费 token

### 解决方案讨论

| 任务类型 | 推荐方案 | 原因 |
|---------|---------|------|
| AI 密集型（写代码/分析/报告） | sessions_spawn 子 agent | 需要 AI 判断力，完成自动通知 |
| 机械耗时型（下载/文件处理） | exec background + 状态文件 | 纯机械操作，不需要 AI，成本低 |
| 需要人类交互 | 主会话处理 | 需要上下文和判断 |
