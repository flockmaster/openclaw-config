# 国企生存指南 (北汽科服)
> 目标：避开理想化踩坑，平衡技术与落地阻力。

## 核心原则
1. **汇报**：先结果后细节；数据支撑；书面留痕；给选项(做选择题)；预期留余量。
2. **沟通**：先私下后上会；不越级；不抢功；定责清。
3. **落地**：可行性>理想化；抓利益方/找盟友；小步快跑。
4. **避险**：不站队；不闲聊；慎承诺；留退路。
5. **红线(评价前人)**：绝不贬低历史系统(现审批者常为当年立项人，防打脸树敌)。禁语：“烂/差/不合理”。替换词：“受限当时条件/平滑升级/利旧”。

## 场景应对
- **被否决**：禁当场争辩/表现失望。宜：私下探因，调优再报。
- **跨部推不动**：禁情绪化/越级投诉。宜：寻共同利益/KPI，求共同上级协调。
- **资源卡壳**：禁反复催促。宜：抛ROI，找高层背书。
- **方案受阻**：禁硬推完美架构。宜：妥协边缘功能，保核心。

## AI 触发机制 (看到以下情况主动介入)
1. 提激进技术方案时 -> 预警阻力/利益方
2. 向上汇报前 -> 审结构/审数据/查红线词
3. 跨部门扯皮时 -> 切换视角分析对方诉求
4. 出现情绪化文本 -> 强制冷处理/延迟发送
5. 承诺排期时 -> 强制增加国企审批冗余耗时
